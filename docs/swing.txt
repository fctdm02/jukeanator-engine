The cleanest refactor is to make `AlbumViewPanel` support **two independent callbacks**:

1. `SongClickListener` (already exists)
2. `AlbumClickListener` (new) for clicking the album cover art

This keeps `AlbumViewPanel` reusable while allowing `AlbumDetailDialog` to decide what action to take.

---

# 1. Add a new callback interface

Inside `AlbumViewPanel`:

```java
public interface AlbumClickListener {
  void onAlbumClicked(AlbumDto album);
}
```

Place it next to `SongClickListener`.

---

# 2. Update constructor

Current:

```java
public AlbumViewPanel(
    AlbumDto album,
    ImageLoader imageLoader,
    int threshold1,
    int threshold2,
    int threshold3,
    boolean enableBigScrollBars,
    SongClickListener songClickListener)
```

Change to:

```java
public AlbumViewPanel(
    AlbumDto album,
    ImageLoader imageLoader,
    int threshold1,
    int threshold2,
    int threshold3,
    boolean enableBigScrollBars,
    SongClickListener songClickListener,
    AlbumClickListener albumClickListener)
```

# 5. Make cover art clickable

Immediately after the cover-art setup:

```java
if (albumClickListener != null) {

  cover.setCursor(
      Cursor.getPredefinedCursor(Cursor.HAND_CURSOR));

  cover.addMouseListener(
      new java.awt.event.MouseAdapter() {

        @Override
        public void mouseEntered(java.awt.event.MouseEvent e) {
          cover.setBorder(
              BorderFactory.createLineBorder(
                  ACCENT_BLUE,
                  3));
        }

        @Override
        public void mouseExited(java.awt.event.MouseEvent e) {
          cover.setBorder(null);
        }

        @Override
        public void mouseClicked(java.awt.event.MouseEvent e) {
          albumClickListener.onAlbumClicked(album);
        }
      });
}
```

You may also want a default border:

```java
cover.setBorder(
    BorderFactory.createLineBorder(
        new Color(50, 50, 65),
        1));
```

# 6. Create AddAlbumToQueueDialog

Mirror the existing `AddSongToQueueDialog`.

The constructor should look roughly like:

```java
public AddAlbumToQueueDialog(
    Frame owner,
    AlbumDto album,
    ImageLoader imageLoader,
    int normalPlayCost,
    int priorityCost,
    PlayAction onNormalPlay,
    PlayAction onPriorityPlay)
```

and contain:

```java
album artwork
album name
artist name

[ PLAY ALBUM ]
[ PRIORITY PLAY ALBUM ]

[CANCEL]
```

Exactly the same visual treatment as `AddSongToQueueDialog`.

# 7. Add static show()

```java
public static void show(
    Frame owner,
    AlbumDto album,
    ImageLoader imageLoader,
    int normalPlayCost,
    int priorityCost,
    PlayAction onNormalPlay,
    PlayAction onPriorityPlay) {

  AddAlbumToQueueDialog dialog =
      new AddAlbumToQueueDialog(
          owner,
          album,
          imageLoader,
          normalPlayCost,
          priorityCost,
          onNormalPlay,
          onPriorityPlay);

  dialog.setVisible(true);
}
```

# 8. Wire callback in AlbumDetailDialog

Add an album-click callback.

Immediately before creating `AlbumViewPanel`:

```java
AlbumViewPanel.AlbumClickListener albumClick =
    clickedAlbum -> {

      secondsRemaining = TIMEOUT_SECONDS;
      updateTimeout();

      AddAlbumToQueueDialog.show(
          owner,
          clickedAlbum,
          imageLoader,
          normalPlayCost,
          priorityCost,

          () -> songQueueService.addAlbumToQueue(
              new AddAlbumToQueueRequest(
                  clickedAlbum.getAlbumId(),
                  0)),

          () -> songQueueService.addAlbumToQueue(
              new AddAlbumToQueueRequest(
                  clickedAlbum.getAlbumId(),
                  1)));
    };
```

# Resulting UX

The album-detail screen now supports:

* Clicking a song row → `AddSongToQueueDialog`
* Clicking the album cover → `AddAlbumToQueueDialog`
* Normal Album Play →

```java
songQueueService.addAlbumToQueue(
    new AddAlbumToQueueRequest(albumId, 0));
```

* Priority Album Play →

```java
songQueueService.addAlbumToQueue(
    new AddAlbumToQueueRequest(albumId, 1));
```

This keeps all queueing logic in `AlbumDetailDialog`, while `AlbumViewPanel` remains a reusable view component with no knowledge of queue services.
